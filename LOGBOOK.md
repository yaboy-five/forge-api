# 📘 FoodForge Development Logbook

## Project Overview

FoodForge is a full-stack food ordering application developed as part of a Software Development workplace simulation. The project consists of a React frontend and an Express backend that communicate through RESTful APIs.

## Technologies Used

- React (Vite)
- Node.js
- Express.js
- MongoDB (development)
- Axios
- JWT Authentication
- bcrypt
- Morgan
- Winston
- Git & GitHub
- Postman

---

## Development Timeline

### Week 1
- Set up frontend and backend repositories.
- Configured Express server.
- Added Morgan and Winston middleware.
- Created initial routes and controllers.
- Implemented error handling.
- Resolved MongoDB connection and package installation issues.

### Week 2
- Improved routing architecture.
- Developed menu and cart functionality.
- Integrated Axios for API communication.
- Performed unit and integration testing.
- Resolved MongoDB and OneDrive synchronization issues.

### Week 3
- Built React pages for Menu, Cart, Checkout and Status.
- Connected frontend to backend APIs.
- Improved application structure.
- Fixed routing and import/export errors.

### Week 4
- Completed remaining page functionality.
- Implemented service files.
- Added Home page.
- Researched and planned JWT authentication.
- Updated backend controller functionality.
- Began documentation.

### Week 5
- Implemented user authentication.
- Created Login and Register pages.
- Added AuthContext and protected routes.
- Completed frontend UI.
- Tested API endpoints using Postman.
- Finalized documentation and prepared project for submission.
